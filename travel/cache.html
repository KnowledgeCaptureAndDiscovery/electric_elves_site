Using Cache Capabilities

  • Default Set Up
    1. Create a Database. You can use any ODBC compatible DB. For creating an Access database go to:
      • Start/Programs/Microsoft Access
      • Create a "Blank Access Database", and name it (for example "mydb.mdb").
    2. Create an ODBC Data Source for the newly created database. Go to:
      • Start/Settings/Control Panel/Administrative Tools/Data Sources(ODBC)
      • Choose "MS Access Database", and say "Add"
      • Choose "MS Access Driver", and say "Finish"
      • Enter the Data Source name, actcache, choose "Select Directory", and navigate to the directory where you created your empty database. Select the newly created DB, "mydb.mdb" as the database.
      • Choose "OK", and you are done
    3. By default, images are downloaded in the directory HERACLES_HOME/cache.
  • Custom Set Up
    1. Choose from the Heracles "Cache" menu "Choose Cache Folder". This will set the caching folder AND will create an empty database in that folder.
    2. Create an ODBC Data Source for the newly created database. Go to:
      • Start/Settings/Control Panel/Administrative Tools/Data Sources(ODBC)
      • Choose "MS Access Database", and say "Add"
      • Choose "MS Access Driver", and say "Finish"
      • Enter the Data Source name, actcache(or anything else), choose "Select Directory", and navigate to the directory chosen in Step 1. Select the newly created DB, "heracles.mdb" as the database.
      • Choose "OK", and you are done
    3. Choose from the Heracles "Cache" menu "Enter DataSource Name", and enter the Data Source name specified at the previous step (if it is not actcache).
    4. Now the images will be downloaded in the folder chosen at step 1, and the database used for caching is the database that was created in this folder.
  • Clearing the Cache

    By selecting from the Heracles "Cache" menu "Clear Cache", all contents of the current download directory will be removed, and all contents of the database stored in that directory will be removed.

You can specify whether you want to use this database capabilities or not in the application's File menu. By default, data is retrieved from the wrappers, and if the wrapper fails it is retrieved from the DB if it exists in the DB.
